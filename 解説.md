# agi-lab_farm_agent — 作品解説

## 1. この作品は何か

**農業の現場で「どこで・何を育てているか」だけを入力すると、気象・衛星（植生）・（必要なら）マイクロ波由来の土壌・降水情報を一度に集め、ブラウザで開ける一枚の HTML ダッシュボードにまとめるツール**です。あわせて、**Claude Code 上で同じコードベースを触るときの手引き**を Skill としてプラグイン化しています。

線形のパイプライン実行に加え、**OpenAI のツール呼び出しで「どのデータをいつ取るか」をモデルに任せるモード**（`--llm-agent`）も選べます。

---

## 2. 誰の、どんな手間を減らすか

| 想定する利用シーン | これまで起きがちなこと | この作品の狙い |
|-------------------|------------------------|----------------|
| 圃場の状況を短時間で把握したい | 気象サイト・地図・衛星ツールを別々に開く | **1 回のコマンド**で要約と可視化をそろえる |
| 光学衛星だけでは弱い季節・天候 | 雲で NDVI が取りづらい | **土壌水分プロキシ**や **JAXA（降水・SMC）** で補う選択肢 |
| AI エージェントの「自律的な取得順」を見せたい | パイプラインが固定だと説明しづらい | **LLM がツール順を選ぶ**デモが可能 |

最終的な栽培判断は本ツールだけでは代替できません。**現地確認と専門家の判断を優先する**前提の「材料集め」と位置づけています。

---

## 3. データの流れ（ざっくり）

1. **ジオコーディング**  
   住所・地名、または `緯度,経度` を緯度経度に変換します（Nominatim 利用時は利用ポリシーに注意）。

2. **気象**  
   Open-Meteo から、指定期間の降水・気温などの要約を取得します。

3. **NDVI（植生指標）**  
   Google Earth Engine 上の Sentinel-2 を用い、代表点周辺のバッファで中央値合成などの集計を行い、**平均 NDVI** とシーン数などを出します。

4. **土壌水分（補助）**  
   NDVI が弱い・欠測に近い場合など、再解析ベースの**土壌水分プロキシ**で補完する流れがあります。

5. **JAXA（任意）**  
   `jaxa-earth` を入れ、`--jaxa` や環境変数で有効にすると、**GSMaP の日次降水**と **AMSR2 の土壌水分（SMC）** を同じ期間・矩形で取りに行き、HTML の **JAXA 用カード**に載せます。公式の「サムネ URL」はないため、**ラスタを PNG にして HTML に埋め込む**方式です（matplotlib、`FARM_AGENT_JAXA_THUMB` でオンオフ）。

6. **提案文**  
   ルールベースで、作物・気象・NDVI・土壌・JAXA の有無に応じた**参考文案**を Markdown 風に生成し、HTML の「管理提案」に表示します。

7. **出力**  
   **`farm_agent/reports/latest.html`** に、地図（OpenStreetMap 埋め込み）、各カード、グラフ・サムネが入った単一ファイルを書き出します。外部 CDN に依存しない自己完結型です。

---

## 4. 画面（ダッシュボード）の見方

- **青系のカード** … **Sentinel-2 / GEE の NDVI** セクション。衛星由来の植生の目安です。
- **橙系のカード** … **JAXA Earth API** で取った降水・土壌水分のセクション。データソースが一目で区別できるようにしています。

LLM モードを使った場合は、**どのツールをどの順で呼んだか**の一覧も HTML に出ます（デバッグ用の環境変数で NDVI をわざと失敗させるオプションなどもあり、挙動確認に使えます）。

---

## 5. Claude Code の Plugin / Skill の役割

- **marketplace**（`agi-lab-farm-agent`）から **farm-dashboard-plugin** をインストールすると、**`SKILL.md`** がエージェントの参照知識として入ります。
- Skill には、**リポジトリ構成**、**CLI の典型例**、**南緯・西経で argparse が誤解しない書き方**（`--location` / `--crop` や `--` の位置）、**主要ファイルのマップ**、**秘密情報を書かない**といったルールがまとめてあります。
- プラグインに含まれるのは **Skill 用ディレクトリだけ**で、**Python 本体は `farm_agent/` にある**ため、利用者は **リポジトリ全体をクローン**したうえで `farm_agent` フォルダで CLI を実行する想定です。

## 7. まとめ

**「場所と作物」から、気象・光学衛星・（任意で）JAXA までを一枚の HTML に束ね、あわせて Claude Code で開発・説明しやすい Skill を配布する**構成です。デモ動画では、ターミナルでの実行からブラウザでのスクロールまで見せると、体験が伝わりやすくなります。
